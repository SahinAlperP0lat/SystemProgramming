This document outlines the mandatory contribution workflow and professional standards for the Student Information System with Docker Containers project. All team members must adhere to these rules to ensure individual and technical success.
A. General Mandatory Rules

    Account Setup: You must use your official school email for both GitHub and Docker Hub accounts.

    Plagiarism: Copying code from other projects is strictly prohibited and results in disqualification.

    Deadlines: All weekly check-ins and the final submission deadline (Jan 09, 2026) must be met.

B. Communication and Management

    Technical Discussions: Use GitHub Issues for all technical coordination.

    Task Tracking: Tasks must be updated and tracked via GitHub Projects.

    Weekly Meetings: Hold at least one meeting per week and document notes in the GitHub Wiki.

C. Development Workflow

    Branch Strategy: Use main (stable), develop (integration), and feature/* (active development) branches.

    Push Restriction: Direct push to main and develop is strictly prohibited.

    Pull Requests: Every change requires a Pull Request (PR) with at least one review from a teammate.

    Commit Message Format: Follow the "feat/fix: description" style with detailed bullet points and issue links.

        Example: feat: add student insert functionality - Implement insert_student() - Closes #45.

D. Minimum Activity Expectations (Per Role)

To satisfy the 50-point individual evaluation, members must achieve the following:
Activity Type	Student 1 (PM)	Student 2 (DevOps)	Student 3 (Dev)
Meaningful Commits	20+	15+	20+
Opened Issues	10+	8+	8+
Pull Requests	8+	10+	12+
Code Reviews	15+	10+	10+
Wiki Edits	15+	5+	5+
